● Task 3 is done. Here's what was implemented:

  READ_EXPR (2000–2060)
  - Skips whitespace, dispatches on ( → READ_LIST, otherwise collects characters
   into TK$ until a delimiter, then calls INTERN

  READ_LIST (2100–2150)
  - Iterative loop via GOTO — avoids using the BASIC GOSUB stack for list-level
  recursion
  - Before each element's GOSUB 2000, saves RH%/RT% (head and tail pointers)
  onto S1%/S2% stack and restores them after — this is essential because a
  nested list element will overwrite those variables
  - Builds the cons list left-to-right by patching the cdr of the tail cell
  (CD%(-RT%)=RV%) rather than reversing

  The design means nesting depth (e.g. (A (B (C)))) uses the BASIC GOSUB stack,
  while list breadth (e.g. (A B C D ...)) iterates — the right tradeoff for C64
  BASIC.
